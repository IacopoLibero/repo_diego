Come usare GitHub

GitHub è una piattaforma di hosting per il controllo di versione e la collaborazione.
Permette di lavorare insieme su progetti da qualsiasi luogo.
Ecco i passaggi fondamentali per iniziare:
  1. Creare un account: Vai su github.com e registrati per un nuovo account.
  2. Creare un nuovo repository: Dopo aver effettuato l'accesso, clicca sul pulsante "New" per creare un nuovo repository. Dai un nome al tuo repository e scegli se renderlo pubblico o privato.
  3. Configurare il repository locale:
    1. Se hai già del codice locale e/o non è presente su GitHub:
      1. Inizializza il repository locale con git init.
      2. Aggiungi l'origine remota con git remote add origin [URL del repository].
      3. Pushare i cambiamenti iniziali con git push origin master.
    2. Se non hai codice locale ma è presente su GitHub:
      1. Clona il repository con git clone [URL del repository].
      2. Oppure, se il repository esiste già in locale, aggiorna con git pull.
  4. Aggiungere e committare cambiamenti: Usa git add . per aggiungere i file da committare modificati e git commit -m "..." per salvare i cambiamenti con un messaggio descrittivo.
  5. Pushare i cambiamenti: Usa git push origin master per inviare i tuoi cambiamenti al repository remoto su GitHub.
  6. Collaborare: Puoi collaborare con altri utenti creando pull request, che permettono di discutere e rivedere i cambiamenti prima di unirli al branch principale.
  7. Gestire i branch: I branch ti permettono di lavorare su diverse versioni del tuo progetto contemporaneamente.
    1. Creare un nuovo branch senza spostarti nel nuovo branch: Usa git branch [nome-branch] per creare un nuovo branch senza spostarti su di esso.
    2. Creare un nuovo branch: Usa git checkout -b [nome-branch] per creare e spostarti su un nuovo branch.
    3. Visualizzare i branch esistenti: Usa git branch per vedere tutti i branch disponibili.
    4. Unire un branch: Usa git merge [nome-branch] per unire un branch al branch corrente.
    5. Eliminare un branch: Usa git branch -d [nome-branch] per eliminare un branch che non ti serve più.
  8. Fare un fork di un repository: Un fork è una copia di un repository che ti permette di sperimentare con i cambiamenti senza influenzare l'originale.
    1. Vai alla pagina del repository che vuoi forkare su GitHub.
    2. Clicca sul pulsante "Fork" in alto a destra.
    3. Seleziona il tuo account o organizzazione dove vuoi creare il fork.
    4. Clona il fork sul tuo computer con git clone [URL del fork].

Per ulteriori dettagli, consulta la documentazione ufficiale di GitHub.
